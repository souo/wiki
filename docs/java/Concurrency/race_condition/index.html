竞态条件与临界区

当两个线程竞争同一资源时，如果对资源的访问顺序敏感，就称存在竞态条件。 导致竞态条件发生的代码区称作临界区。

两种竞态条件​

  • Read-modify-write
  • Check-then-act

Read-modify-write 竞态条件​

读-修改-写 临界区会导致产生竞态条件，下面是一个例子：

  public class Counter {

     protected long count = 0;

     public void add(long value){
         this.count = this.count + value;
     }
  }

想象下线程A和B同时执行同一个Counter对象的add()方法。 我们无法知道操作系统何时在两个线程之间切换。 add() 方法中的代码不是被Java 虚拟机作为单个原子指令执行的。 而是 作为一组较小的指令执行，类似于：

  • 从内存中读取count的值到寄存器上
  • 增加cpu寄存器中的值
  • cpu寄存器中的值写回内存

观察以下线程 A 和 B 会发生什么：

       this.count = 0;

   A:  Reads this.count into a register (0)
   B:  Reads this.count into a register (0)
   B:  Adds value 2 to register
   B:  Writes register value (2) back to memory. this.count now equals 2
   A:  Adds value 3 to register
   A:  Writes register value (3) back to memory. this.count now equals 3

这两个线程想要将值 2 和 3 添加到计数器中。 因此，在两个线程完成执行后，该值应该是 5。 但是，由于两个线程的执行是交错的，结果最终会有所不同。

Check-then-act 竞态条件​

如果两个线程 检查相同的条件，满足条件之后修改条件，这可能导致竞态条件。

public class CheckThenActExample {

    public void checkThenAct(Map<String, String> sharedMap) {
        if(sharedMap.containsKey("key")){
            String val = sharedMap.remove("key");
            if(val == null) {
                System.out.println("Value for 'key' was null");
            }
        } else {
            sharedMap.put("key", "value");
        }
    }
}

防止竞态条件​

为了防止发生竞态条件，您必须确保临界区作为原子指令执行。这意味着一旦一个线程正在执行它，在该线程离开临界区之前，其他线程都不能执行它。 可以通过在临界区进行适当的线程同步来避免竞争条件。线程同步可以使用Java synchronized block 来实现, 也可以使用其他同步构造（如锁）或原子变量（如 java.util.concurrent.atomic.AtomicInteger）来实现。

临界区吞吐量​

对于较小的临界区，使整个临界区成为同步块可能会起作用。 但是，对于较大的临界区，将临界区分成较小的临界区, 以允许多个线程执行每个较小的临界区。这可以减少对共享资源的争用，从而增加总临界区的吞吐量。
