Spark Join Strategies

Broadcast Hash Join​

先来了解一下Hash Join，大体来说分为两个阶段：


hash join

  • 第一步 基于关联表中的小表，使用既定的哈希函数应用到join key上构建哈希表。
  • 第二步 遍历大表中每一条数据记录使用相同的hash函数 在第一步构建的哈希表中去匹配。

在spark中，hash join作用在节点上, 在每个分区完成关联。 我们来看看 Broadcast Hash Join

BHJ中 Spark 把关联关系同相对较小的表封装到广播变量，然后在全网进行分发, 而体量较大的表只要“待在原地、保持不动”，就能轻松地完成关联计算， 这减少了shuffle的消耗。

当其中一张关联表的大小小于阈值（默认10M）时，spark会选择 BHJ 作为 join 策略，可以通过下面的参数调整阈值大小。

spark.sql.autoBroadcastJoinThreshold
注意事项
  • 仅支持等值连接
  • 支持除全连接外(inner, left, right)的所有join类型

Shuffle hash join​

如果采用 Shuffle 的分发方式来完成数据关联，那么外表和内表都需要按照 Join Key 在集群中做全量的数据分发。因为只有这样，两个数据表中 Join Key 相同的数据记录才能分配到同一个 Executor 进程，从而完成关联计算。

注意事项
  • 仅支持等值连接
  • join key 不需要是可排序的
  • 支持所有join类型
  • 这是一种即需要shuffle又需要hashing的一种连接, 维护哈希表需要计算和内存资源

Shuffle sort-merge join​

我们先来看看 sort-merge join

SMJ 的思路是先排序、再归并。具体来说，就是参与 Join 的两张表先分别按照 Join Key 做升序排序。然后，SMJ 会使用两个独立的游标对排好序的两张表完成归并关联。

再来理解spark中的 Shuffle sort-merge join：

Shuffle sort-merge join涉及到对数据进行shuffle，得到同一个worker的相同join_key，然后在worker节点的partition级别进行sort-merge join操作。

注意事项
  • 仅支持等值连接
  • join key 需要是可排序的
  • 支持所有join类型

Cartesian Join​

在这个策略中，计算两个表的笛卡尔积（类似于 SQL）执行 join。

注意事项
  • 支持等值连接和非等值连接
  • 只支持 inner join类型

Broadcast nested loop join​

使用两个嵌套的 for 循环依次扫描外表和内表中的数据记录，判断关联条件是否满足。

for record_1 in relation_1:
  for record_2 in relation_2:
    # join condition is executed
注意事项
  • 仅支持等值连接和非等值连接(>=, < 等)
  • 支持所有join类型，实现做了如下优化
    • 如果是 right outer join 广播左边
    • 如果是 left outer, left semi, left anti 或者 existence join 广播右边
    • 如果是 inner join 则两边都广播
    • 其他情况下需要扫描数据多次

spark如何选择 join 策略​

从spark代码 来分析spark是如何选择join策略的

  • 如果是等值连接
    • 按以下顺序查看 join hits：
      1. Broadcast Hint: 选择 broadcast hash join 如果是支持的join类型，如果两边都有broadcast hints 选择较小的一边来广播。
      2. Sort merge hint: 选择 sort merge join 如果key是可排序的
      3. shuffle hash hint: 选择 shuffle hash join
      4. shuffle replicate NL hint: 选择 cartesian product 如果join类型是 inner join。
    • 如果没有hint或者hint不适用
      1. 选择 broadcast hash join 如果一边小到足以广播，并且join类型不能是全连接
      2. 选择 shuffle hash join 如果一边小到足以构建本地hashMap (数据分片的平均大小小于设定的阈值)，并且大表至少是小表的3倍大小， 而且spark.sql.join.preferSortMergeJoin is false`
      3. 选择 sort merge join 如果key是可排序的
      4. 选择 cartesian product 如果是内连接
      5. 选择 broadcast nested loop join 作为最终的方案，这可能会OOM，但也只能如此。
